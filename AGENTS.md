# AGENTS.md

이 저장소는 Codex 스킬 `ktt`를 관리한다.

Repository-local 지침이므로 이 저장소 안에서 작업할 때는 아래 규칙을 우선한다.

## 프로젝트 구조

- `SKILL.md`는 런타임 지침만 담는다. 길게 설명하지 말고 명령 판별, 모듈 로드, 매핑, 입력/출력 규칙만 유지한다.
- `MAKE.md`는 새 페르소나 모듈을 만드는 가이드다.
- `references/thread-common.md`는 공통 시스템 레이어다. 모듈 로드, 입력 해석, 정보 수집, 교차 검증, 원문 보호만 담당한다.
- `references/thread-common.md`에는 말투, 줄바꿈, 훅, 반전, 결론 방식 같은 페르소나 문체 규칙을 넣지 않는다.
- `references/thread-*.md` 중 `thread-common.md`를 제외한 파일은 모두 페르소나 모듈이다.
- `README.md`는 사용자용 설명 문서다. 런타임 세부 규칙을 길게 중복하지 않는다.

## 페르소나 모듈 규칙

- 페르소나 호출명은 한글 1단어로 정한다.
- 페르소나 파일명은 영어 단어 또는 한글 발음 로마자로 정한다.
- 새 페르소나는 `references/thread-<module-name>.md` 형식으로 추가한다.
- 새 페르소나를 추가하면 `SKILL.md`의 페르소나 매핑을 갱신한다.
- 새 페르소나를 추가하면 `README.md`의 페르소나 수, 모듈 구조, 모듈 역할 표, 말투 예시를 갱신한다.
- 새 페르소나 생성 절차는 반드시 `MAKE.md`를 따른다.

## 원문 보호

- 사용자가 제공한 텍스트나 이미지 속 문장, 문구, 농담, 비유, 고유 표현을 샘플이나 문서에 그대로 재사용하지 않는다.
- 일부 단어만 바꾼 문장도 사용하지 않는다.
- 개인, 계정, 회사, 커뮤니티를 직접 떠올리게 하는 표현은 피한다.
- 참고 자료는 분석 재료로만 사용하고, 샘플은 다른 테크 주제로 새로 작성한다.

## 정보 수집과 검증

- 정보성 글은 공식 또는 신뢰 가능한 출처를 확인한다.
- 날짜, 버전, 수치, 성능, 가격, 정책, 지원 범위는 교차 검증한다.
- 가능하면 공식 출처 1개 이상을 먼저 확인한다.
- 공식 출처가 없으면 신뢰 가능한 출처 2개 이상으로 확인하고 단정하지 않는다.
- 사용자 입력에 포함된 수치나 고유명사도 그대로 믿지 말고 확인한다.
- 확인한 범위와 확인하지 못한 범위를 구분한다.

## 설치본 동기화

사용자가 이 컴퓨터에서 바로 스킬을 쓰려는 경우, 변경된 파일을 설치본에도 동기화한다.

설치 경로:

```text
C:\Users\Administrator\.codex\skills\ktt
```

주로 동기화할 파일:

- `SKILL.md`
- `README.md`
- `MAKE.md`
- `references/thread-common.md`
- 새로 추가하거나 수정한 `references/thread-*.md`

## 검증

문서나 모듈을 수정한 뒤 가능한 범위에서 다음을 실행한다.

```text
git diff --check
```

새 페르소나를 추가한 경우:

```text
rg "<페르소나명>|thread-<module-name>" -n
```

확인할 것:

- `SKILL.md`의 페르소나 매핑이 실제 파일과 일치하는가?
- `README.md`의 페르소나 수가 실제 개수와 맞는가?
- `thread-common.md`에 페르소나 문체 규칙이 들어가지 않았는가?
- 새 샘플이 제공 자료의 문장이나 고유 표현을 재사용하지 않았는가?
- 정보성 내용이면 공식 또는 신뢰 가능한 출처로 검증했는가?
