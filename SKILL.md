# Korean Tech Threader

## 목적

이 스킬은 사용자가 제공한 URL, 텍스트, 메모, 초안, 아이디어를 한국어 테크 스레드 게시물로 변환한다.

모든 페르소나는 같은 공통 목표를 공유한다.

- 첫 문장에서 관심을 끌기
- 모바일에서 읽히는 짧은 리듬 만들기
- 하나의 중심 주장으로 정리하기
- 중간에 반전 또는 관점 전환 넣기
- 주제에 맞는 핵심 문제나 긴장 요소를 현실적으로 다루기
- 사실과 추정을 구분하기
- 마지막에 기억나는 결론 남기기

페르소나는 이 공통 구조 위에 말투, 화법, 관점만 덧입힌다.

## 호출 형식

다음 짧은 호출 형식을 지원한다.

```text
$ktt <페르소나명>
<URL 또는 내용>
```

예시:

```text
$ktt 실무자
https://example.com/article
```

```text
$ktt 실무자
AI 자동화는 모델 성능보다 운영 비용과 실패 복구가 더 중요하다.
```

## 목록 명령

사용자가 다음처럼 입력하면 스레드를 작성하지 말고 페르소나 목록을 출력한다.

```text
ktt 목록
```

또는:

```text
$ktt 목록
```

목록 출력에는 다음을 포함한다.

- 페르소나명
- 모듈 파일
- 한 줄 설명
- 3문장 안팎의 짧은 예시글

예시글은 실제 사용자 입력이나 참고 자료를 쓰지 말고, 각 페르소나의 말투를 보여주는 새 예시로 작성한다.
예시글은 페르소나당 3문장 정도만 보여준다.

현재 목록 출력 대상:

- `실무자` -> `references/thread-practitioner.md`
- `해커` -> `references/thread-hacker.md`

목록 출력 형식:

```text
사용 가능한 페르소나

| 페르소나 | 모듈 | 특징 |
|---|---|---|
| 실무자 | references/thread-practitioner.md | 운영 현실, 실패 복구, 비용 구조를 중심으로 보는 실무자 말투 |
| 해커 | references/thread-hacker.md | 작은 도구 조합, 빠른 실험, 비용 절감을 짧게 던지는 말투 |

예시

[실무자]
AI 기능은 붙이는 것보다 운영하는 게 더 어려움.
처음엔 작은 자동화처럼 보이는데, 실패 복구가 없으면 바로 부담이 커짐.
결국 중요한 건 똑똑한 답변보다 망가졌을 때 돌아오는 구조임.

[해커]
자동화는 처음부터 크게 만들 필요 없다.
작은 기록 하나 두고 실행 결과만 남기면 된다.
핵심은 도구가 아니라 상태를 잃지 않는 구조다.
```

## 모듈 로드 순서

스레드를 작성할 때는 반드시 다음 순서로 모듈을 적용한다.

1. `references/thread-common.md`를 읽고 공통 스레드 작성 규칙을 적용한다.
2. `$ktt` 다음 첫 토큰을 페르소나명으로 해석한다.
3. 페르소나 매핑에 따라 영어 모듈 파일을 찾는다.
4. `references/thread-<module-name>.md` 파일을 읽고 말투와 화법을 적용한다.
5. 페르소나명이 없거나 일치하지 않으면 `실무자`를 기본값으로 사용한다.
6. 공통 규칙과 페르소나 규칙이 충돌하면 공통 규칙을 우선한다. 단, 말투와 화법은 페르소나 규칙을 따른다.

현재 페르소나 매핑:

- `실무자` -> `references/thread-practitioner.md`
- `해커` -> `references/thread-hacker.md`

## 입력 해석

`$ktt` 다음 첫 토큰이 페르소나 매핑에 있으면 해당 영어 모듈 파일을 적용한다.

나머지 텍스트는 원문 자료로 취급한다.

해당 페르소나 매핑이 없으면 `실무자`를 적용하고, `$ktt` 뒤의 전체 내용을 원문 자료로 취급한다.

입력이 URL이면 원문 내용을 확인하고, 필요한 경우 추가 리서치로 사실 여부를 검증한다.

입력이 메모나 초안이면 하나의 중심 주장으로 재구성한다.

## 출력

사용자가 별도 형식을 요구하지 않으면 설명 없이 완성된 스레드 본문만 출력한다.

여러 버전을 요청하면 각 버전을 명확히 구분하되, 각 스레드는 독립적으로 읽히게 만든다.

검증이 필요한 정보가 불확실하면 사실처럼 단정하지 말고 완화 표현을 사용한다.
